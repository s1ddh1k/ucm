## Design Context

### Users
Solo developer building an open-source autonomous agent runtime. Primary user is the creator; future audience is open-source developers who want mission-driven automation. Users are technical, opinionated, and expect tools that feel authoritative — not friendly, not playful.

### Brand Personality
**Strict. Complete. Perfect.**

UCM is not an IDE. It is not a chat app. It is a runtime that takes a mission and produces a deliverable without human intervention. The interface must reflect that discipline: every element earns its place, nothing is decorative, nothing is approximate.

The emotional goal is **trust** — the user should feel the system is in control and operating with precision.

### Aesthetic Direction
- **Visual tone**: Clinical, precise, restrained. Think instrument panel, not dashboard widget. Closer to Dieter Rams than Material Design.
- **Color**: Cool and neutral. No warm tones — warm is explicitly rejected. Use slate, zinc, or neutral grays as the foundation. Accent colors should be cold and functional (blue-gray, teal, or muted cyan for signals — never amber, orange, or copper).
- **Theme**: Dark mode is primary. Light mode is secondary. Both must feel equally considered.
- **Typography**: Clean sans-serif. Maintain Korean font support (Apple SD Gothic Neo, Malgun Gothic). Tight, purposeful hierarchy.
- **Anti-references**: Must NOT look like Eclipse, VSCode, JetBrains, Codex app, or Claude app. No IDE chrome, no sidebar tree panels, no chat bubbles, no tab bars that look like browser tabs. UCM is a different category of tool and must look like it.
- **What it IS like**: A mission control console. A surgical instrument. A system that demands respect through restraint.

### Design Principles
1. **Earn every pixel** — If an element doesn't serve the mission-to-deliverable path, remove it.
2. **Trust through precision** — Alignment, spacing, and typography must be exact. Sloppiness erodes trust.
3. **Status, not decoration** — Color communicates state (running, blocked, completed, failed). It is never ornamental.
4. **Korean-first text** — All UI copy is bilingual (ko/en). Korean is the primary language; layouts must accommodate Korean text naturally, not as an afterthought.
5. **Dark by default** — Dark mode is the primary surface. Light mode is a considered alternative, not a desaturation of dark.
